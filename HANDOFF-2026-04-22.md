## P0 — Cohérence IA : objectif 99 % de réponses justes & reproductibles

### Constat
Le bench `--repeat 3` (2026-04-22, agent TVA, 10 questions) montre :
- **9/10 PASS** sur le critère factuel (contains/regex)
- **4/10 instables** : similarité textuelle entre runs `<0.55` ou écart-type score `>0.15`
- Une même question donne parfois trois reformulations très différentes alors que le fond est identique

L'utilisateur veut **99 % de réponses justes ET reproductibles**. Aujourd'hui Qwen3.5-35B est appelé avec `temperature=0.2` + `enable_thinking=false` → assez stable mais pas assez.

### Pistes à explorer (dans cet ordre)

1. **Baisser drastiquement la stochasticité** — premier essai gratuit
   - `temperature` : 0.2 → 0.0
   - `top_p` : 0.9 → 0.1
   - `top_k` : 40 → 1 (greedy)
   - `repeat_penalty` : 1.1 → 1.0 (sinon le modèle peut diverger pour éviter répétition)
   - Lancer `run_quality.py --agents tva conseiller cloture --repeat 5` avant/après pour mesurer
   - **Risque** : trop greedy → réponses très répétitives, peut dégrader sur questions ouvertes

2. **Self-consistency (majority vote N=3)**
   - Lancer 3 appels en parallèle, comparer les réponses, retenir celle qui apparaît majoritairement (ou agréger les chiffres clés)
   - Coût : ×3 latence et tokens, mais si parallèle on garde la même latence end-to-end
   - Implémenter dans `LexaAgent.ask()` derrière un flag `USE_SELF_CONSISTENCY`

3. **Two-stage retrieve → format**
   - Étape 1 : agent « factual » qui extrait les chiffres clés du RAG en JSON strict (plafond, taux, articles cités)
   - Étape 2 : agent « writer » qui formate ce JSON en prose
   - L'instabilité tombe à zéro sur les chiffres ; reste de la diversité dans la prose mais c'est OK
   - Le RAG existe déjà (Qdrant `swiss_law` 9761 points), il faut juste un schéma JSON strict en sortie de l'extracteur

4. **Cache déterministe de réponses**
   - Hash `(question, profile, year, top RAG hits)` → cache Redis 7j
   - Pour les questions FAQ (3e pilier plafond, taux TVA, etc.) on retombe sur la même réponse
   - Existe déjà `LlmQueue` dans le backend — voir si extensible

5. **Structured output via JSON Schema**
   - vLLM/Qwen3 supportent `response_format: json_schema` (OpenAI-compat). Pour les questions « combien / quel taux / plafond » on force un JSON `{ answer_chf: number, citations: [{law, article}], explanation: string }`
   - Templater côté backend pour produire la phrase finale → 100 % déterministe sur les chiffres

### À livrer

- POC self-consistency (2) sur LexaAgent uniquement, mesure cohérence
- Décision : self-consistency suffit, ou il faut two-stage (3) ?
- Documenter dans `tasks/coherence-99pct.md` la méthode retenue

## P1 — Bugs UX bloquants

### B1 — Import Swigs Pro ne fonctionne pas
**Symptôme** : depuis `Settings → Intégrations Pro`, le bouton « Synchroniser maintenant » échoue ou ne ramène rien.
- Backend route : `POST /settings/integrations/pro/sync` (`apps/backend/src/routes/settings.ts`)
- Bridge Pro→Lexa : `apps/backend/src/routes/bridge.ts` + Hub Pro côté `swigs-workflow`
- Vérifier : token Hub valide ? `hubUserId` mappé sur `users.hub_user_id` ? logs PM2 `lexa-backend` au moment du clic
- Penser à tester avec un user qui A des factures dans Pro (pas tous les comptes)

### B2 — OCR images (certificat salaire PNG/JPG) → timeout
**Symptôme** : upload PNG/JPG dans le modal PP → timeout côté frontend (≥120s ?).
- Pipeline : `apps/backend/src/services/OcrExtractor.ts` → vLLM Qwen3-VL côté `.103`
- Vérifier : Qwen3-VL est-il vraiment up sur `.103:???` ? Est-ce qu'il y a un fallback PDF qui ne marche pas pour images ?
- `pm2 logs lexa-backend` pendant un upload PNG montrera l'étape qui timeout
- PDF fonctionne (test-cert-salaire.pdf OK) → le problème est probablement spécifique à `image/png` / `image/jpeg`

### B3 — Wallet crypto : bouton refresh sans effet
**Symptôme** : ajout wallet OK, clic « Refresh snapshot » ne déclenche rien visuel.
- Frontend : `apps/frontend/src/components/workspace/v2/PpCryptoSwimlane.tsx` ou `PpCryptoWalletForm.tsx` — vérifier le handler `onRefresh`
- Backend : `POST /pp/crypto/snapshot/refresh` retourne `{jobId, estimatedSeconds}` — le frontend doit ensuite poll `GET /pp/crypto/snapshot?year=YYYY`
- Probablement : pas de polling, ou pas de toast/spinner → l'UX donne l'illusion que rien ne se passe alors que le backend bosse
- À fixer : ajouter spinner « Snapshot en cours… » + auto-poll toutes les 5s pendant 60s + invalider la query React Query

  3. Supprimer le composant + ses entry points + son route éventuelle
