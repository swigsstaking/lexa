## Découvertes importantes

1. **Le prototype existant est d'une qualité remarquable.** Score 97% sur grille d'évaluation, méthodologie scientifique (grille détaillée, comparaison Q4/Q8, dataset audité). Pas une exploration bâclée — du vrai travail d'ingénieur.

2. **La plupart des fondations techniques existent déjà** : Qdrant + BGE-M3 + modèle spécialisé + XML Fedlex officiels + dataset de fine-tuning + OCR service. On bâtit un produit par-dessus, on ne repart pas de zéro.

3. **Le fine-tuning LoRA est bloqué par une contrainte hardware** : torch+CUDA non dispo sur aarch64 (DGX Spark GB10). Le script `finetune.py` est prêt mais doit tourner sur une machine x86. À décider : est-ce qu'on fait le fine-tuning (grappiller 3% supplémentaires) ou est-ce que le system prompt + RAG suffisent déjà (97% est excellent) ?

4. **La base légale actuelle est uniquement fédérale** (LIFD + LTVA + CO). **Aucun cantonal encore**, **LHID manquante** (pourtant fondamentale pour le droit cantonal). Premier gros chantier de KB.

5. **Swiss GAAP RPC** (référentiel comptable) est payant donc non ingérable automatiquement. À acheter en format structuré ou à lire manuellement pour enrichir les agents.

6. **Les 3 processus Ollama actifs en prod** sur le Spark servent à d'autres projets (consulting). Il faudra bien documenter ce qu'utilise Lexa vs ce qui est partagé pour éviter les conflits.
