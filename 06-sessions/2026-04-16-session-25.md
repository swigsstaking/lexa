## Bloc A — Fix pipeline OCR

### Bugs corrigés

**Bug 1 — `pdf-parse@1.1.1` / `1.1.4` incompatible avec pdfkit** :  
`pdf-parse` lève `bad XRef entry` sur les PDF générés par pdfkit. Vérifié sur prod : logs montrent `[ocr] pdf-parse failed: bad XRef entry`.

**Bug 2 — Ollama `images[]` refuse PDF brut** :  
Le code original envoyait le buffer PDF brut en base64 après l'échec de pdf-parse. Ollama retourne HTTP 500 sur les PDF bruts.

### Solution implémentée

`pdfToPng(pdfBuffer: Buffer): Promise<Buffer>` — conversion PDF → PNG via `pdfjs-dist@4.10.38` (legacy/build/pdf.mjs, disableWorker) + `@napi-rs/canvas@0.1.67`. Rendu à scale 2.0 (≈200 dpi). Aucun binaire système requis.

Pipeline `extractRawText` :
1. PDF → essai `pdf-parse` (text.length > 50) → si OK : `method=pdf-parse, confidence=0.95`
2. PDF → si pdf-parse échoue ou texte court → `pdfToPng()` → Ollama vision
3. Image directe → Ollama vision

**Fix bonus** : `parseOcrModelOutput()` rendue récursive via `flattenJsonToText()` — gère les JSON imbriqués multi-niveaux que qwen3-vl-ocr produit (ex: `{"Rémunération": {"Salaire brut": "CHF 85'000.00"}}`).

### Smoke test E2E manuel

Upload `test-cert-salaire.pdf` via `POST /api/documents/upload` :
```json
{
  "documentId": "7abbdcd1-6dbe-4ca1-9a33-14ff2d9190a2",
  "ocrResult": {
    "rawText": "CERTIFICAT DE SALAIRE 2025\n...\nSalaire brut annuel : CHF 85'000.00\n...",
    "extractionMethod": "pdf-parse",
    "ocrConfidence": 0.95,
    "type": "certificat_salaire",
    "extractedFields": {
      "employer": "Lexa Test SA",
      "employeeName": "TEST Jean",
      "grossSalary": 85000,
      "netSalary": 72500,
      "year": 2025,
      "period": "01.01.2025 - 31.12.2025",
      "avsLpp": 6420
    },
    "durationMs": 5876
  }
}
```

Résultat : `type=certificat_salaire`, `grossSalary=85000` (≥80000 requis) ✓  
Les deux chemins (pdf-parse success + pdfToPng fallback) ont été validés dans les logs.

### Décision : GARDER qwen3-vl-ocr

`deepseek-ocr` (version custom sur le DGX Spark) retourne du contenu vide (output empty) sur les images PNG — il est inutilisable pour l'OCR. qwen3-vl-ocr : 100% précision, texte brut déterministe, 0 échec sur 3 itérations, ~22s stable.

**`deepseek-ocr` ne sera pas utilisé en production.** La latence plus courte (1.7s) est illusoire — le modèle ne retourne rien d'utile.
