## Bloc A — Ingestion KB Canton Vaud ✅

### A.0 — Recherche source HTML VD (Subagent Explore + investigation manuelle)

**Problème découvert** : contrairement à GE (SILGeneve = HTML statique legacy ASP.NET), le Canton Vaud n'expose pas de HTML statique direct. `prestations.vd.ch` est une **SPA Angular** qui ne sert que `<app-root>` en HTML brut.

**Sources testées** :
- `rsv.vd.ch` → DNS inaccessible depuis le Spark
- `prestations.vd.ch/pub/blv-publication/actes/consolide/642.11` → SPA Angular, `<app-root>` uniquement
- Fedlex → SPA également pour le cantonal
- lexfind.ch → SPA Angular

**Solution trouvée (investigation manuelle ~45 min)** :
En analysant le `main.js` de la SPA Angular BLV, j'ai trouvé l'API REST backend exposée par la SPA :
1. `GET api/actes/CONSOLIDE?id={acte_id}&cote={cote}` → JSON avec les versions de l'acte, dont la version ACTUELLE et son `htmlId`
2. `GET api/actes/{htmlId}/html` → XHTML AkomaNtoso **UTF-8** complet de la loi

**IDs BLV (version 2026)** :
| Loi | acte_id | htmlId (version ACTUELLE) |
|-----|---------|--------------------------|
| LI 642.11 | `8df99d51-8df8-49ed-9ef5-2fa4817d0004` | `a28b9d22-658e-4236-b5b1-95eccc04d2b1` |
| LIPC 650.11 | `66dcd6f6-b573-4422-a30f-b0d0845151cf` | `0848120c-c688-409b-9bf7-fe174efd0bfc` |
| RLI 642.11.1 | `5390e38d-2c99-4168-8778-9828e56c0009` | `afe66749-8119-4719-9912-b86b21f4a944` |

Firefox Playwright installé sur le Spark (`python3 -m playwright install firefox`) pour découvrir l'API via interception réseau. Une fois l'API trouvée, le scraping est du HTTP simple sans Playwright.

**Source retenue** : `prestations.vd.ch/pub/blv-publication/api/actes/{htmlId}/html` — HTML statique AkomaNtoso UTF-8 ✅

### A.1 — Script `ingest_vd_laws_lexa.py`

Clone structurel de `ingest_ge_laws_lexa.py` avec 3 différences clé :

1. **Fetch HTTP** : API REST JSON BLV (`api/actes/CONSOLIDE` pour résoudre le htmlId dynamiquement + `api/actes/{id}/html` pour le contenu). UTF-8 direct, pas de windows-1252.
2. **Parser AkomaNtoso** : `class="akn-article-container"` → `class="akn-num"` (numéro) + `class="akn-heading"` (titre) + `class="akn-alinea"` (alinéas dans le sibling `div.collapse`). Regex `clean_heading` pour supprimer les notes de bas de page `[1, 2, 3]`.
3. **Métadonnées payload Qdrant** (identiques aux clés GE sauf canton/jurisdiction) :
   - `jurisdiction: "cantonal-VD"`, `canton: "VD"`, `topic: "loi_fiscale_vd"`, `source: "prestations.vd.ch"`
   - `law ∈ {LI-VD, LIPC-VD, RLI-VD}`
