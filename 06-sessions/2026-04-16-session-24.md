## Bloc A — DocumentMapper + apply route + field-sources

### A.1 — Analyse schémas existants

Découverte critique : le schéma draft **n'utilise pas** `income.salary` (comme dans le plan initial) mais les vrais paths sont :
- `step2.salaireBrut` pour le salaire brut (certificat_salaire → grossSalary)
- `step4.pilier3a` pour le 3ème pilier (attestation_3a → amount)

La table `taxpayer_drafts` utilise `fiscal_year` (pas `year`) et n'a pas de colonne `canton` (le canton est dans `state.step1.canton` ou dans `taxpayer_profiles`).

### A.2 — DocumentMapper.ts

Créé : `apps/backend/src/services/DocumentMapper.ts`

Pure function, zéro side effect :
- `certificat_salaire` → `{ fieldPath: "step2.salaireBrut", value: grossSalary }`
- `attestation_3a` → `{ fieldPath: "step4.pilier3a", value: amount }`
- Autres types → `[]` (pas d'erreur, juste mapping vide)

Export : type `FieldMapping` + fonction `mapDocumentToFields`.

### A.3 — Route apply-to-draft

Ajoutée dans `apps/backend/src/routes/documents.ts` :

```
POST /documents/:id/apply-to-draft
Body: { year: 2026 }
→ { ok: true, fieldsApplied: ["step2.salaireBrut"], message: "1 champ(s) pré-rempli(s)" }
```

Logique :
1. Lookup document (isolation tenant)
2. Lookup draft Postgres par `fiscal_year` (pas de création automatique — 404 si absent)
3. `mapDocumentToFields(...)` → mappings
4. `structuredClone(state)` + `setDeep(newState, path, value)` pour chaque mapping
5. UPDATE `taxpayer_drafts.state`
6. `$push` dans `documents_meta.appliedToDrafts` (provenance)
7. Event Postgres `DocumentAppliedToDraft` (streamId = documentId si UUID valide, sinon `randomUUID()`)

Correction bug : `streamId` dans EventStore doit être UUID. Si le documentId n'est pas un UUID (documents synthétiques de test), on génère un UUID pour le streamId.

### A.4 — Route field-sources

Ajoutée dans `apps/backend/src/routes/taxpayers.ts` :

```
GET /taxpayers/draft/:year/field-sources
→ { "step2.salaireBrut": { documentId, filename, appliedAt } }
```

Logique : join Mongo (appliedToDrafts.draftId) + reverse map fieldPath → source la plus récente.

### A.5 — Event type DocumentAppliedToDraft

Ajouté dans `apps/backend/src/events/types.ts` :
```ts
{ type: "DocumentAppliedToDraft"; payload: { documentId, draftId, fiscalYear, fieldsApplied } }
```
