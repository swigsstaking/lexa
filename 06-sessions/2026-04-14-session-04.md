## Décisions structurelles prises

1. **Pattern Playwright validé** pour les sites Cloudflare + Angular/JS. lex.vs.ch passe sans problème. Même pattern utilisable pour les cantons suivants (GE, VD, FR, NE, JU, BE-Jura) s'ils utilisent lexfind.ch ou un système similaire.

2. **Cache local systématique** pour les ressources coûteuses à fetch : PDFs, XML, textes HTML. Permet de re-parser sans retélécharger.

3. **Regex-based parser pour les textes rendus** (innerText) est une approche viable pour les sites Angular — ne pas s'appuyer sur le DOM (instable), utiliser la structure textuelle.

4. **Scripts "clone and patch" plus efficaces qu'un script générique** : pour chaque nouveau type de document, cloner un script similaire et adapter. Moins d'abstractions prématurées, plus facile à déboguer.

5. **Limitation acceptée** : le parser VS a extrait 175/285 articles détectés. Pas parfait mais suffisant pour le MVP. Amélioration à planifier.

## Découvertes importantes

1. **Playwright passe Cloudflare sans effort** sur `lex.vs.ch`. Le headless Chromium + user-agent standard suffit. Pas besoin de plugins anti-bot (playwright-stealth) ni de résoudre un captcha. Solution réutilisable pour tous les sites cantonaux protégés.

2. **Les portails JSF AFC (gate.estv.admin.ch) sont un chantier complexe** : pas juste un scraping, il faut cliquer section par section, la langue est coincée en allemand, les `ViewState` changent à chaque requête. **Ne pas sous-estimer le temps nécessaire**.

3. **Le RAG retourne les bons documents en fonction de la spécificité** : le Guide PP VS 2024 dépasse souvent la Loi fiscale VS sur les queries pratiques, parce qu'il contient des chiffres, exemples, et formulations concrètes. La loi reste la référence juridique, mais pour une utilisation pratique, les guides/directives sont plus utiles. C'est exactement ce qu'on veut : l'agent Conseiller consultera d'abord les guides, et l'agent Audit ira à la loi pour la citation légale formelle.

4. **Les URLs Fedlex XML ne se devinent pas par pattern trial** — il faut passer par la page HTML de la loi pour trouver le vrai ELI path. Mon approche "boucle HEAD sur patterns" a trouvé LP mais pas OIFD/OLTVA. À généraliser : scraper la page HTML Fedlex du document pour extraire l'URL XML canonique.

5. **Le password sudo partagé dans le chat est une erreur opsec** — même dans un environnement privé. À la prochaine session, demander au user d'utiliser `ssh` interactif lui-même pour les tâches sudo, ou de pousser le password dans une variable d'environnement temporaire sur la machine cible.
