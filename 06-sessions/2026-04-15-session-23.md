## Bloc B — OcrExtractor + routes

### OcrExtractor.ts

- Stage 1 : `pdf-parse` pour PDFs texte (fallback si texte < 20 chars → vision)
- Stage 1 image/scanned : `qwen3-vl-ocr` via Ollama `/api/chat`
- **Piège critique** : `qwen3-vl-ocr` retourne du JSON non-déterministe en contenu de message :
  - Format A : `{"text": "..."}` — texte dans champ text
  - Format B : `{"text": [...]}` — tableau de lignes
  - Format C : `{"ligne1": "", "ligne2": ""}` — clés = lignes OCR, valeurs vides
  - Résolution : fonction `parseOcrModelOutput()` qui gère les 3 cas
- Stage 2 : `qwen3.5:9b-optimized` format JSON strict, prompt en français
- 4 types documentaires + fallback `autre`
