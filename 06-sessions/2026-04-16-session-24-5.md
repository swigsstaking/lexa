## Problèmes identifiés (dette technique)

### 1. pdf-parse incompatible avec PDFKit (impact prod élevé)

`pdf-parse@1.1.1` lève `bad XRef entry` sur les PDF générés par PDFKit.
**Impact** : En production, les clients qui uploadent des PDF générés par des logiciels
courants (Apple Numbers, LibreOffice, PDFKit-based tools) tomberont sur le fallback vision
même si le PDF contient du texte embarqué — résultats moins fiables et plus lents.

**Solution recommandée** : migrer vers `pdfjs-dist` ou `pdf2json`.

### 2. Ollama images[] = PNG/JPEG uniquement

Confirmé empiriquement : Ollama refuse les PDF base64 dans `images[]` avec HTTP 500.
OcrExtractor.ts doit toujours s'assurer de convertir les PDF en image avant d'appeler Ollama.
La conversion pdf-parse (stage 1 texte embarqué) contourne ce problème pour les PDF valides,
mais le fallback vision actuel est cassé pour les PDF PDFKit (problème 1 + 2 combinés).
