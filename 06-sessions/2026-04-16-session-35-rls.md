## Bloc B — Exemples migration avant/après

### Avant
```ts
// EventStore.ts
import { query } from "../db/postgres.js";
const result = await query<EventRow>(
  `SELECT * FROM events WHERE tenant_id = $1 AND stream_id = $2`,
  [tenantId, streamId],
);
```

### Après
```ts
// EventStore.ts
import { queryAsTenant } from "../db/postgres.js";
const result = await queryAsTenant<EventRow>(
  tenantId,
  `SELECT * FROM events WHERE tenant_id = $1 AND stream_id = $2`,
  [tenantId, streamId],
);
```

### Fix critique — postgres.ts wrapper

Le wrapper S32 utilisait `SET LOCAL` avec paramètre (non supporté Postgres) :
```ts
// AVANT (cassé)
await client.query("SET LOCAL app.active_tenant = $1", [tenantId]);

// APRÈS (correct)
await client.query("SELECT set_config('app.active_tenant', $1, true)", [tenantId]);
```
