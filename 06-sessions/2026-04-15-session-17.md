## Incidents / pièges rencontrés

1. **Backend tourne via tsx watch src/** : le rsync du dist ne sert à rien, il faut synchroniser les sources `.ts`. Coût : 15 min.
2. **Templates YAML dans src/ pas dist/** : avant la mise au jour, dist/execution/templates/ était vide, créait un ENOENT silencieux. Résolu en copiant les YAMLs VS dans dist et en syncant src/.
3. **Import `query` manquant** : les routes profile utilisaient `query` sans l'importer → erreur TS 2304. Résolu en ajoutant `import { query } from "../db/postgres.js"`.
4. **`civilStatus` type union strict** : le profil DB retourne `string`, le schema step1 attend une union. Cast avec validation VALID_CIVIL[].
