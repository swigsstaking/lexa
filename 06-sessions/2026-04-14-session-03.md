## Décisions structurelles prises

1. **Stack PDF sur le Spark** : `pdfplumber` + `pymupdf` dans `~/.local/lib/python3.12/site-packages/` (avec `--break-system-packages`), exécution via `python3` système (pas dans le venv `ft-env`). Le venv était un cul-de-sac par l'absence de `FlagEmbedding`.

2. **Pattern d'ingestion Lexa** : **un script `ingest_<type>_lexa.py` par type de document** (lois XML, circulaires PDF, webpublications HTML, etc.). Pas de script monolithique. Chaque script est additif (UUID4, pas de `delete_collection`).

3. **Chunking PDF** : heuristique 2-étapes (détection sections numérotées avec fallback paragraphes). Max 2000 chars par chunk pour rester dans la limite `max_length=512` de BGE-M3 avec une marge.

4. **Format payload enrichi** pour les circulaires AFC : ajout de `category`, `page`, `source`, URL du PDF. Pour le cantonal VS : ajout de `jurisdiction` pour permettre le filtrage cantonal dans les requêtes RAG.

5. **Infos TVA modernes (HTML)** : reportées session 04 avec un scraper dédié. Les webpublications `gate.estv.admin.ch` ne sont pas en PDF téléchargeable.

6. **Loi fiscale VS intégrale** : reportée session 04+ avec un scraper Playwright/headless Chrome (Cloudflare bloque curl). En attendant, le Guide déclaration 2024 + les directives 2025 donnent une couverture opérationnelle correcte.

7. **BGE-M3 sur Spark** : **tourne en CPU uniquement** (torch+CUDA indispo sur aarch64 GB10). Pas bloquant mais ralentit les ingestions. À noter pour les estimations de temps.

## Découvertes importantes

1. **Nomenclature AFC changée depuis 2010** — les anciens numéros "Info TVA 01/03/14" n'existent plus tels quels. Ma proposition initiale session 02 était basée sur une nomenclature obsolète. **Toujours vérifier via un subagent web avant de se lancer dans une ingestion.**

2. **Cloudflare sur lex.vs.ch bloque les bots** — pour les sites cantonaux protégés, il faudra un scraper Playwright/headless Chrome. À prévoir dans l'outillage de session 04+.

3. **Les webpublications AFC sont en HTML, pas PDF** — depuis 2010, l'AFC publie ses Info TVA sous forme de pages HTML interactives sur `gate.estv.admin.ch`. Il faudra un scraper BeautifulSoup ou utiliser le bouton "créer PDF" du portail. C'est un nouveau chantier en session 04.

4. **BGE-M3 tourne en CPU sur le Spark GB10** — même problème que le fine-tuning LoRA : pas de torch+CUDA aarch64. BGE-M3 fonctionne mais est **lent** (~1.25 chunks/s en CPU vs <16 chunks/s sur GPU). Pour ~1400 chunks = 18 min. Pas bloquant mais à prévoir pour les grosses ingestions futures (7 cantons × ~500 chunks = ~1.5h d'encodage pur).

5. **Le PEP 668 bloque `pip` en system** sur Ubuntu récent. Solution Lexa : `--break-system-packages` dans le site user, pas dans un venv (pour garder l'accès à `FlagEmbedding` qui est déjà dans `~/.local/`).

6. **La collection Qdrant `swiss_law` a un état partagé** — les 791 points initiaux + 108 LHID + ~1400 AFC + ~100 VS = ~2400 points. Il faut maintenir la discipline UUID4 strict pour ne jamais écraser les points existants.
