**Méthode d'ingestion** : flow JSF en 4 étapes qui génère un PDF complet en français :
1. GET `tableOfContent.xhtml?publicationId=X` → récupérer cookies + ViewState
2. POST changement de langue en français (ID `j_idt20:j_idt24` sur la page)
3. GET `pdfSelectionList.xhtml?publicationId=X&lang=fr` → nouveau ViewState + N checkboxes
4. POST avec toutes les checkboxes cochées → retourne directement `application/pdf` (FR)

Script : `~/ollama-compta/scripts/ingest_afc_info_tva_lexa.py`

## Format de payload Qdrant (pour les circulaires PDF)

```yaml
text: "[<label>] <texte du chunk>"
law: "AFC-IFD-Circ-<num>"  # ou "AFC-Notice-<code>"
law_label: "<titre complet>"
article: "section <idx>"  # index séquentiel du chunk dans le document
article_num: "<idx>"
heading: "<première ligne du chunk, max 100 chars>"
rs: null  # les circulaires n'ont pas de RS
topic: "<thème>"
date_version: "<date d'entrée en vigueur>"
source: "afc"
category: "circular-ifd" | "notice"
page: <numéro de page source dans le PDF>
url: "<URL officielle du PDF>"
```

## Méthodologie de chunking

Les PDFs AFC ne sont pas structurés en articles comme les lois fédérales. Le script `ingest_afc_pdfs_lexa.py` utilise une heuristique en 2 temps :

1. **Détection de sections numérotées** (`1.`, `1.1`, `Art. X`, `Chapitre Y`, etc.) via regex
2. **Fallback chunking par paragraphes** si moins de 3 sections détectées (max 2000 chars par chunk)

Chaque chunk garde une référence à sa page d'origine pour permettre au modèle de citer précisément (`Circ IFD 26, section 3.2, p. 4`).
